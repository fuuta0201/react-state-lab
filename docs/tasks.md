# ベーシックなタスク管理アプリ 実装タスク

## 前提

- 対象アプリは React + TypeScript + Vite + Tailwind CSS で実装する。
- 管理する項目は、タイトル、説明、作成日時、ステータス、優先度とする。
- 初期実装ではバックエンドを持たず、ブラウザ内の state と `localStorage` で管理する。
- 各機能・各コンポーネントは、実装と同じタイミングで Storybook とユニットテストを追加する。

## 進め方

各作業単位で、次の 3 点をまとめて完了させる。

- [ ] 実装
- [ ] Storybook
- [ ] ユニットテスト

Storybook が不要な純粋関数や型定義は、Storybook の代わりに利用コンポーネント側の Story で確認する。

## 実装順序

1. データモデルを作成し、テストする。
2. タスク操作ロジックを作成し、テストする。
3. 日付処理、永続化、絞り込み、並び替えの純粋関数を作成し、テストする。
4. `Badge`、`StatusSelect`、`PrioritySelect`、`EmptyState` を作成し、Storybook とテストを追加する。
5. `TaskForm` を作成し、Storybook とテストを追加する。
6. `TaskItem` と `TaskEditor` を作成し、Storybook とテストを追加する。
7. `TaskList` と `TaskFilterBar` を作成し、Storybook とテストを追加する。
8. `TaskApp` で全体を接続し、Storybook と統合寄りのテストを追加する。
9. `App.tsx` から `TaskApp` を表示する。
10. アクセシビリティ、レスポンシブ、全体動作を確認する。
